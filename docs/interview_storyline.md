# Interview Storyline

## 90-second opening

"I created this project as a portfolio-safe simulation of how I would run a Microsoft 365 Copilot adoption program in a large enterprise. Instead of showing a generic chatbot, I focused on the work this role actually does: readiness assessment, use case prioritization, training and champions, adoption analytics, value measurement, and executive facilitation."

## 5-minute walkthrough

### 1. Explain the scenario

"This is a synthetic global enterprise rollout across finance, legal, customer service, and executive staff. I chose those personas because they each have different value cases, guardrails, and change needs."

### 2. Show readiness and role fit

"The dashboard starts with a readiness score and direct mapping to the job requirements. I wanted the reviewer to see the consulting logic first, not the UI first."

### 3. Show one use case

"For example, the finance close sprint is not just a prompt library. It has an owner, a wave, success goals, and guardrails."

### 4. Run the planner

"The planner takes an interview scenario and turns it into a plan with readiness steps, training actions, support actions, value actions, and objection experiments."

### 5. Show support, facilitation, and packet preview

"I included a tiered support model, a facilitation board, and a rollout packet checker because this role is not only about adoption metrics. It is also about removing blockers, keeping decisions moving, and making sure sponsors see a clear, reviewable launch packet."

## What to emphasize technically

- deterministic backend logic instead of hand-wavy AI magic
- structured data models for readiness, value, and facilitation
- product thinking in the frontend
- enterprise-safe truthfulness boundary

## Closing line

"What I wanted to demonstrate here is that I can connect business outcomes, change management, user enablement, and practical technical execution in one coherent adoption story."
